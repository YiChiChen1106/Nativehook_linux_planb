# native_hook 入门复习

## 1. 这个项目是干什么的

一句话：

`native_hook` 是一个内存采集工具，用来观察某个进程的内存申请和释放行为，并把数据传给后台服务做分析。

可以先把它理解成：

`业务进程 -> native_hook -> 共享内存 -> 后台服务 -> trace 文件`

## 2. 关键名词

### malloc

- `malloc` = 申请一块新的内存
- 返回值可以理解成这块内存的“门牌号”
- 用完后一般要 `free`

例子：

```c
char *p = malloc(100);
free(p);
```

### free

- `free` = 释放之前申请过的堆内存
- hook `free` 的意义是记录“哪块内存什么时候被释放了”
- 这样才能把“申请”和“释放”配对起来，分析泄漏和内存存活时间

### mmap

- `mmap` = 映射一块内存区域
- 常见用途：
  - 文件映射
  - 匿名内存
  - 大块内存管理
- hook `mmap` 的意义是补足 `malloc/free` 看不到的内存行为

### 调用栈

- 调用栈 = “是谁一层层调用到这里的”
- 它回答的是：
  - 这次内存申请是谁触发的
  - 它是从哪条函数链路来的

例子：

`页面加载 -> 图片解码 -> 缓存分配 -> malloc`

### 采样

- 采样 = 不是每次都记，只挑一部分记
- 目的是降低性能开销

常见思路：

- 每隔一段采一次
- 只采大对象
- 小对象直接跳过

### 共享内存

- 共享内存 = 两个程序共用的一块“白板”
- 一个进程写，另一个进程读
- 比每次都写文件或频繁发消息更快

在这个项目里可以理解成：

- `native_hook` 先把数据写入共享内存
- 后台服务再从共享内存里取走数据

## 3. 为什么这个工具可能会慢

因为它工作在高频路径上。

像 `malloc/free/mmap` 这种操作，程序可能会调用很多很多次。  
如果每次都做很多额外动作，就容易带来明显性能开销。

常见额外动作包括：

- 判断要不要采
- 抓线程信息
- 抓调用栈
- 拷贝栈数据
- 写共享内存
- 通知后台服务

## 4. 热点路径

- 热点路径 = 会被反复执行很多次的代码路径
- 在这个项目里，`malloc/free/mmap` 的 hook 路径通常就是热点路径

重要原则：

不要在热点路径上做重活。

## 5. 同步开销 / 异步开销

### 同步开销

- 同步开销 = 多线程为了不打架而等待、排队、抢锁带来的成本
- 典型来源：
  - 互斥锁
  - 自旋锁
  - 多线程共享同一份数据

一句话理解：

大家都抢同一个公共资源，就会慢。

### 异步开销

- 异步 = 先把事情记下来，稍后再处理
- 好处是主流程更快
- 但把任务交给后台也不是零成本，仍然会有：
  - 入队
  - 通知
  - 数据拷贝
  - 后台处理

一句话理解：

异步比当场做完更轻，但也不是免费。

## 6. 批量上报

- 批量上报 = 不来一条发一条，而是攒一批再发

好处：

- 减少通知次数
- 减少 flush 次数
- 减少后台被频繁唤醒

一句话理解：

一条条寄快递很浪费，攒一包再寄更省。

## 7. 为什么抓调用栈会贵

因为抓调用栈不是只记一个数字，而是要沿着当前执行路径往上追很多层。

它通常会涉及：

- 找线程栈范围
- 一层层回溯栈帧
- 记录更多数据

所以规律是：

- 栈越深，通常越贵
- 记录越全，通常越贵

## 8. 为什么 <1% 很难

因为这是在高频热点路径上做采集。

如果要求：

- 高频 `malloc/free` 全抓
- 每次都抓调用栈
- 栈还抓得很深

那开销通常很难非常低。

更现实的目标往往是：

- 默认推荐配置下
- 通过采样、过滤、浅栈、批量上报
- 尽量把开销压低

## 9. 常见优化词汇

### thread local

- `thread local` = 每个线程自己有一份私有数据
- 好处是减少线程之间抢同一个数据

一句话理解：

每个线程一个小抽屉，不用抢公共抽屉。

### 对象池

- 对象池 = 不要每次都创建和销毁临时对象，而是重复使用现成对象

一句话理解：

不要每次新买杯子，用完放回去下次继续用。

### 读写锁

- 读写锁 = 读可以并行，写必须独占
- 适合“读很多、写很少”的共享数据

一句话理解：

看资料的人可以一起看，改资料的人要单独改。

## 10. 看性能优化建议时的翻译方法

以后看到建议时，可以这样翻译：

- 减少锁竞争
  - 降低同步开销
- 改成 thread-local
  - 避免大家抢同一份状态
- 用对象池
  - 减少热路径上的临时对象创建销毁
- 缓存栈范围
  - 减少重复工作
- 批量 flush / 批量上报
  - 减少通知次数和唤醒次数
- 限制栈深 / 提高采样间隔 / 提高过滤阈值
  - 让进入重逻辑的次数更少

## 11. 你现在最重要的记忆卡片

- `malloc`：申请新内存
- `free`：释放旧内存
- `mmap`：映射一块内存区域
- `调用栈`：这次内存操作是谁触发的
- `采样`：不是每次都抓，只抓一部分
- `共享内存`：采集端和服务端共用的数据白板
- `热点路径`：会被重复执行很多次的路径
- `同步开销`：抢锁、等待带来的成本
- `异步开销`：交给后台处理也有成本
- `批量上报`：攒一批再发，减少频繁通知

## 12. 一句话总复习

`native_hook` 是一个在高频内存路径上工作的采集工具。  
它通过 hook `malloc/free/mmap` 观察内存行为，通过调用栈定位来源，通过采样和批量上报降低开销，通过共享内存把数据交给后台服务。
