# Work Log 2026-06-05

## 一、Sub-stage 36 死锁修复

**根因**：`hook_writer.cpp:RecordStackWriterSubAblationAllocThreadLocal` 中：
```
stack_writer_.Lock()    → pthread_mutex_lock(&inner_mutex_)
  stack_writer_.Write() → pthread_mutex_lock(&inner_mutex_)  // 同一线程重入非递归锁 → 死锁
```

**修复**：commit `e19571b` — 删除外部 `Lock()/Unlock()` 调用（4 行），`Write()` 内部自行管理锁。

**验证**：21/21 单元测试通过，pink 冒烟测试 sub=36 不再 hang。

## 三、分析

**逐层拆解**（1T 为例）：
- Ring write + 内锁 (sub=34): 0.28s
- + eventfd 通知 (sub=35): 0.82s → delta 0.54s = eventfd syscall 开销
- + consumer drain (sub=36): 0.84s → delta 0.03s = consumer 消费开销（几乎可忽略）

**线程扩展**：
- sub=34（纯锁）：16T/1T = 2.6x — inner_mutex_ 竞争随线程放大
- sub=35（锁+通知）：16T/1T = 1.0x — 几乎平坦，eventfd 无竞争
- sub=36（全链）：16T/1T = 1.0x — consumer drain 也不随线程恶化

**对比 batch64 (sub=33, 旧数据)**：
- batch64 HookWriter: 1T=1.24s, 16T=1.37s
- per-record StackWriter (sub=36): 1T=0.84s, 16T=0.88s
- StackWriter 内锁临界区 << HookWriter 全局 mutex，per-record StackWriter 反而比 batch64 HookWriter 快
